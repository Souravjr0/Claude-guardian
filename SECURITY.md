# Security Policy

## Supported versions

This project is in active development. Security fixes are applied to the latest `main` branch.

## Reporting a vulnerability

Please do not open public issues for suspected vulnerabilities.

Instead, email: `security@souravjr0.dev` with:

- Description of the issue
- Reproduction steps
- Potential impact
- Suggested mitigation (if known)

You can expect an acknowledgment within 72 hours.

## Secrets and credentials

- Never commit real API keys.
- Use `.env` for local secrets.
- Keep `.env.example` sanitized.
